# Impact Mapping 인터뷰 질문

## Phase 1: Goal (Why)

목표를 명확히 하는 질문들.

### 필수 질문

| # | 질문 | 의도 |
|---|------|------|
| G1 | "이 프로젝트의 궁극적인 목표가 뭐야?" | 진짜 목표 파악 |
| G2 | "6개월 후 '성공했다'고 느끼려면 뭐가 달성되어야 해?" | 측정 가능한 기준 |
| G3 | "명시적으로 '이건 목표가 아니다'라고 할 것은?" | Anti-goals (scope 방지) |

### 후속 질문

| 상황 | 질문 |
|------|------|
| 목표가 애매할 때 | "그걸 통해 얻고 싶은 게 뭐야?" |
| 수단과 목표 혼동 | "그건 수단인 것 같은데, 궁극적으로 뭘 원해?" |
| 여러 목표 | "가장 중요한 건 뭐야? 우선순위를 정한다면?" |
| Anti-goals 모를 때 | "이런 것들이 있을 수 있어: [예시 목록]. 이 중 아닌 건?" |

### Anti-Goals 예시 제시용

```
- 커뮤니티 성장 (GitHub stars, contributors)
- 범용 프레임워크 (모든 사람이 쓸 수 있는)
- 완벽한 문서화
- 수익 창출
- 학습 목적 자체
- 특정 기술 습득
```

## Phase 2: Actors (Who)

사용자/이해관계자 파악.

### 필수 질문

| # | 질문 | 의도 |
|---|------|------|
| A1 | "이 시스템의 사용자는 누구야?" | Actor 식별 |
| A2 | "각 사용자가 이 시스템에서 얻고자 하는 가치는?" | 니즈 파악 |
| A3 | "사용자 우선순위가 있다면?" | Primary/Secondary 구분 |

### 후속 질문

| 상황 | 질문 |
|------|------|
| 본인만 사용자 | "혹시 다른 사람도 쓸 가능성은?" |
| 여러 유형 | "가장 중요한 사용자는 누구야?" |
| AI도 사용자 | "AI/에이전트가 이 시스템을 사용할 수도 있어?" |

## Phase 3: Impacts (How)

행동 변화 파악.

### 필수 질문

| # | 질문 | 의도 |
|---|------|------|
| I1 | "이 시스템이 성공하면 너의 작업 방식이 어떻게 바뀔까?" | Impact 정의 |
| I2 | "현재 Pain Point는 뭐야?" | 해결할 문제 |
| I3 | "구체적인 사용 시나리오를 하나 들어줄래?" | 현실적 검증 |

### 후속 질문

| 상황 | 질문 |
|------|------|
| Pain Point 애매 | "지금 그 작업할 때 가장 불편한 게 뭐야?" |
| 시나리오 필요 | "현재는 이렇게 하고, 미래에는 이렇게 하고 싶다 형태로 설명해줄래?" |
| 측정 기준 | "개선 후 어떤 상태가 되면 만족해? 구체적인 숫자로?" |

## Phase 5: Constraints (제약 조건)

### 필수 질문

| # | 질문 | 의도 |
|---|------|------|
| C1 | "비용/시간/기술적 제약이 있어?" | 제약 파악 |


## 인터뷰 진행 팁

### Do's

- 한 번에 2-3개 질문
- 답변 듣고 후속 질문 조정
- 각 Phase 완료 시 요약 확인
- 예시 제시로 답변 유도

### Don'ts

- 한 번에 10개 질문 던지기
- 고정된 질문만 하기
- 기술적 세부사항부터 시작
- 사용자 답변 무시하고 진행

### 인터뷰 마무리

```
"지금까지 정리하면:
- Goal: ...
- Actor: ...
- Impact: ...
- Deliverable: ...
- Constraint: ...

맞아? 수정하거나 추가할 거 있어?"
```
